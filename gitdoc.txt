Setting up a
.gitignore file for your new repository before you get going is generally a good idea so you don’t
accidentally commit files that you really don’t want in your Git repository.
The rules for the patterns you can put in the .gitignore file are as follows:
• Blank lines or lines starting with # are ignored.
• Standard glob patterns work, and will be applied recursively throughout the entire working
tree.
• You can start patterns with a forward slash (/) to avoid recursivity.
• You can end patterns with a forward slash (/) to specify a directory.
• You can negate a pattern by starting it with an exclamation point (!).
Glob patterns are like simplified regular expressions that shells use. An asterisk (*) matches zero or
more characters; [abc] matches any character inside the brackets (in this case a, b, or c); a question
mark (?) matches a single character; and brackets enclosing characters separated by a hyphen ([0-
9]) matches any character between them (in this case 0 through 9). You can also use two asterisks to
match nested directories; a/**/z would match a/z, a/b/z, a/b/c/z, and so on.

# ignore all .a files
*.a
# but do track lib.a, even though you're ignoring .a files above
!lib.a
# only ignore the TODO file in the current directory, not subdir/TODO
/TODO
# ignore all files in any directory named build
build/
# ignore doc/notes.txt, but not doc/server/arch.txt
doc/*.txt
# ignore all .pdf files in the doc/ directory and any of its subdirectories
doc/**/*.pdf

###### ADDING SSH KEY and PUSHING CODE TO REMOTE REPOSITOR ################

=============================================================================================

You’ve managed to hit both classic Git mistakes in one go. Impressive efficiency.

Let’s untangle this properly.

🔴 Problem 1:

remote origin already exists

Translation: Git is telling you
“Stop. You already added this remote. Why are you adding it again?”

✅ Fix:

Check what remote you already have:

git remote -v

If it’s wrong, update it:

git remote set-url origin git@github.com:SugrivArbad/CppPrimerCodes.git

If you really want to nuke it and re-add:

git remote remove origin
git remote add origin git@github.com:SugrivArbad/CppPrimerCodes.git
🔴 Problem 2 (Actual blocker):

Permission denied (publickey)

This is GitHub basically saying:

“Nice try. I don’t know who you are.”

You’re using SSH (git@github.com
) but your machine has no SSH key configured or GitHub doesn’t know about it.

🛠️ Fix SSH Authentication (Do this carefully)
Step 1: Check if you already have SSH keys
ls ~/.ssh

Look for:

id_rsa
id_rsa.pub

or

id_ed25519
id_ed25519.pub

If nothing is there → you’re starting from zero.

Step 2: Generate SSH key
ssh-keygen -t ed25519 -C "your_email@example.com"

Just keep pressing Enter unless you actually care about custom paths.

Step 3: Start SSH agent
eval "$(ssh-agent -s)"

Add key:

ssh-add ~/.ssh/id_ed25519
Step 4: Copy public key
cat ~/.ssh/id_ed25519.pub

Copy that whole thing.

Step 5: Add it to GitHub

Go to:
👉 https://github.com/settings/keys

Add new SSH key → paste it.

Step 6: Test connection
ssh -T git@github.com

Expected:

Hi SugrivArbad! You've successfully authenticated
Step 7: Push again
git push -u origin main
💡 Alternative (if SSH feels like overengineering your life)

Use HTTPS instead:

git remote set-url origin https://github.com/SugrivArbad/CppPrimerCodes.git
git push -u origin main

It will ask for credentials (or token). Less cool, more reliable.

🧠 What actually happened here
You added remote twice → Git got annoyed
You used SSH without setup → GitHub rejected you
Now Git is just standing there like: “figure your identity first”

Get SSH working once and your future self will thank you instead of rage-googling this again at 2 AM.

You don’t “get remote code” with magic. You clone the repo. That’s literally what Git was built for.

Go to the folder where you want your project:

Since you already suffered through SSH setup once, use it:

Create a folder CppPrimerCodes

Set up origin automatically (no drama this time)

You should see:

git init → “I am starting a brand new project”

Use clone when:

You’re starting from scratch
